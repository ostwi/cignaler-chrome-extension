Cignaler connects your browser to the Cignaler desktop app, letting you add CI/CD pipeline watchers with a single click. Currently supports GitLab, with GitHub Actions and Jenkins coming soon.

🔧 HOW IT WORKS

1. Browse any GitLab project page
2. Click the Cignaler icon in your toolbar
3. The extension detects the project name and current branch
4. Click "Add Watcher" to send it to your Cignaler desktop app
5. Cignaler starts monitoring that pipeline — you'll get notified when it finishes

No copy-pasting project paths. No switching windows. One click.

✅ FEATURES

• Automatic detection of project paths and branches/tags
• Works on project pages, pipelines, merge requests, commits, and more
• Supports self-hosted GitLab instances (add your custom hostname in settings)
• GitHub Actions and Jenkins support coming soon
• Customizable watcher names before adding
• Manual branch/tag entry when the extension can't detect one from the URL

⚙️ REQUIREMENTS

• The Cignaler desktop app must be installed on your computer
• The native messaging host must be registered (the Cignaler installer handles this)
• A CI server must be configured in Cignaler — set the matching server name in the extension's options

🔒 PRIVACY

• All communication stays local — the extension talks only to the Cignaler desktop app on your computer via Chrome's Native Messaging
• No data is sent to any external server
• No analytics, tracking, or telemetry
• Settings (CI server name, custom hostnames) are stored locally in Chrome

🛡️ PERMISSIONS

• activeTab — reads the current tab to detect project info when you click the icon
• storage — saves your settings (CI server name, custom hostnames) locally
• nativeMessaging — communicates with the Cignaler desktop app on your machine
• Host permissions (optional) — only requested for self-hosted CI platform domains you add
